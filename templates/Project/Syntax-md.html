Comprehensive Markdown Syntax Guide

This page provides a detailed overview of Markdown syntax. Markdown is a powerful tool for creating formatted text using a simple and readable syntax. It's widely used for writing documentation, notes, and web content. This guide covers a wide range of Markdown elements, from basic formatting to more advanced features.

Headings

Headings structure your content and indicate different levels of importance. Markdown supports six levels of headings.

Markdown:

# Heading 1
## Heading 2
### Heading 3
#### Heading 4
##### Heading 5
###### Heading 6

HTML Output:

Heading 1


Heading 2


Heading 3


Heading 4


Heading 5


Heading 6

Use headings to break up long documents and improve readability. Start with # for the main heading and increase the number of hashes for lower-level headings.

Paragraphs

Paragraphs are the fundamental building blocks of text in Markdown. Separate paragraphs with one or more blank lines.

Markdown:

This is the first paragraph. It can contain multiple sentences and flow naturally.

This is the second paragraph. Notice the blank line separating it from the first one.

HTML Output:

This is the first paragraph. It can contain multiple sentences and flow naturally.



This is the second paragraph. Notice the blank line separating it from the first one.

Markdown automatically wraps text within paragraphs. You don't need to manually insert

tags.

Line Breaks

By default, Markdown ignores single line breaks within a paragraph. To force a line break, end a line with two or more spaces.

Markdown:

This line will have a break  <-- two spaces at the end
right here.

HTML Output:

This line will have a break
right here.

For semantic line breaks (like in addresses or poems), consider using HTML's
tag directly in your Markdown for clarity.

Emphasis: Bold and Italics

Emphasize text using bold and italics to highlight important words or phrases.

Italics

Use single asterisks * or underscores _ around text for italics.

Markdown:

*This text is italicized.*
_This text is also italicized._

HTML Output:

This text is italicized.
This text is also italicized.

Bold

Use double asterisks ** or double underscores __ around text for bold.

Markdown:

**This text is bold.**
__This text is also bold.__

HTML Output:

This text is bold.
This text is also bold.

Bold and Italics Combined

Combine asterisks and underscores to create both bold and italic text.

Markdown:

***This text is bold and italic.***
___This text is also bold and italic.___
**_This text is bold and italic too._**
*_This text is also bold and italic._*

HTML Output:

This text is bold and italic.
This text is also bold and italic.
This text is bold and italic too.
This text is also bold and italic.

For consistency within a document, choose either asterisks or underscores for emphasis and stick with it.

Ordered Lists

Ordered lists are created using numbers followed by a period .. The actual numbers don't need to be sequential in Markdown; it will order them correctly in the HTML output, but it's good practice to keep them sequential for readability in Markdown source.

Markdown:

3. Third item

Links

Create hyperlinks to other web pages or resources. Links consist of link text and the URL.

Markdown:

[Link text to Example Website](https://www.example.com)
[Link to Google](https://www.google.com "Google's Homepage") <-- with title attribute

HTML Output:

Link text to Example Website
Link to Google

Optionally, you can add a title attribute to the link by including it in double quotes after the URL.

Inline Style Links (Reference Links)

Reference style links make Markdown source more readable, especially for long URLs or when links are reused multiple times.

Markdown:

This is [a reference link][link_id].

[link_id]: https://www.example.com "Optional title"

HTML Output:

This is a reference link.

Define the link URL and optional title elsewhere in the document using the [link_id]: URL "Title" syntax. link_id can be any text, and it's case-insensitive.

Images

Embed images into your Markdown documents. The syntax is similar to links, but with an exclamation mark ! at the beginning.

Markdown:

![Alt text for image](image.jpg)
![Logo of Example Website](https://www.example.com/logo.png "Example Logo") <-- with title

HTML Output:

Always provide meaningful alt text for images for accessibility and SEO.

Reference Style Images

Similar to reference links, you can use reference style for images.

Markdown:

![Alt text for reference image][image_ref]

[image_ref]: image.jpg "Image Title"

Code

Inline Code

Use backticks ` to denote inline code snippets, like function names or commands within a sentence.

Markdown:

Use the `` tag for inline code.

HTML Output:

Use the <code> tag for inline code.

Code Blocks (Fenced)

Create code blocks by using triple backticks ``` before and after the code. You can optionally specify a language identifier after the opening backticks for syntax highlighting (depending on the Markdown processor).

Markdown:

```
                function helloWorld() {
                    console.log("Hello, world!");
                  }
                  ``` <-- with language identifier

HTML Output:

function helloWorld() {
                    console.log("Hello, world!");
                  }
                  

                  
                  
function helloWorld() {
                    console.log("Hello, world!");
                  }
                  
 <-- with language class

Code Blocks (Indented)

Alternatively, you can create code blocks by indenting each line of code by at least four spaces or one tab. This method is less common and fenced code blocks are generally preferred.

Markdown:

    This is an indented
                      code block.
                      It requires indentation on each line.

HTML Output:

This is an indented
                  code block.
                  It requires indentation on each line.

Blockquotes

Use blockquotes to quote text from another source. Start each line with a greater than symbol >.

Markdown:

> This is a blockquote.
                  > It can span multiple lines.
                  >
                  > Even with blank lines within.

HTML Output:


                      
This is a blockquote.

                      
It can span multiple lines.

                      
Even with blank lines within.

Blockquotes can contain other Markdown elements, like lists or headings.

Horizontal Rules

Create thematic breaks or visual separation using horizontal rules. Use three or more asterisks *, hyphens -, or underscores _ on a line by themselves.

Markdown:

---
                  ***
                  ___
                  - - -
                  * * *
                  _ _ _

HTML Output:

All of these Markdown syntaxes will produce an

tag in HTML.

Task Lists (Checkboxes)

Create task lists with checkboxes. This is often a feature of GitHub Flavored Markdown (GFM) and other Markdown extensions.

Markdown:

- [x] Task completed
                  - [ ] Task not completed

HTML Output:

    
                      
  •  Task completed
    • 
                      
  •  Task not completed
    •

Use - [x] for checked items and - [ ] for unchecked items. These often render as interactive checkboxes (depending on the Markdown processor).

Tables

Create tables to organize data. Tables are defined using pipes | and hyphens -.

Markdown:

| Header 1 | Header 2 | Header 3 |
                  | -------- | -------- | -------- |
                  | Row 1, Col 1 | Row 1, Col 2 | Row 1, Col 3 |
                  | Row 2, Col 1 | Row 2, Col 2 | Row 2, Col 3 |

HTML Output:


                      
                          

                              Header 1
                              Header 2
                              Header 3
                          

                      
                      
                          

                              Row 1, Col 1
                              Row 1, Col 2
                              Row 1, Col 3
                          

                          

                              Row 2, Col 1
                              Row 2, Col 2
                              Row 2, Col 3

The header row is separated from the body rows by a line of hyphens. Alignment can be controlled in the header separator line using colons :. :--- for left-align, ---: for right-align, and :---: for center-align.

Markdown (with alignment):

| Left-aligned | Center-aligned | Right-aligned |
                  | :----------- | :----------: | ------------: |
                  | Left cell    | Center cell  | Right cell    |
                  | More text    | More text    | More text     |

Strikethrough

Mark text as deleted or no longer relevant using strikethrough. This is often a GFM extension.

Markdown:

This text is ~~strikethrough~~.

HTML Output:

This text is strikethrough.

Use double tildes ~~ around the text to apply strikethrough.

Escaping Characters

To use Markdown syntax characters literally, escape them with a backslash \.

Markdown:

\*This will not be italic\*
                  \*\*This will not be bold\*\*
                  \[This will not be a link](http://example.com)

HTML Output:

*This will not be italic*
**This will not be bold**
[This will not be a link](http://example.com)

You can escape characters like *, _, #, [, ], (, ), `, >, +, -, ., !, |, and \ itself.

HTML in Markdown

Markdown allows you to use raw HTML tags within your Markdown documents. This can be useful for elements not directly supported by Markdown syntax or for more complex layouts. However, using too much HTML can reduce the readability of your Markdown source.

Markdown:

This is a paragraph with blue text using HTML.

HTML Output:

This is a paragraph with blue text using HTML.

Be cautious when using raw HTML as it bypasses Markdown's simplicity and might make your document less portable if you switch Markdown processors.

This guide covers the core and some extended basic Markdown syntax elements. Depending on the specific Markdown processor or flavor (like GitHub Flavored Markdown), you might find additional features and variations. Always refer to the documentation of your specific Markdown implementation for the most accurate and complete syntax details.
